# Course — 课程设计与审查

![Course](https://cdn.jsdelivr.net/gh/kryies/images@main/misc/course-cover.png)

**设计新课程、逐课生成文档、扫描已有课程的质量问题。** 五条通用检查规则 + 八条生成规范，适配任何技术栈。

Design syllabi, generate lessons, and audit course quality. Five universal rules + eight authoring standards. Stack-agnostic.

---

## 命令

```
/course design <课程名>        → 交互式设计课程大纲
/course lesson <课程目录> <N>   → 生成/重写第 N 课
/course init <课程目录>         → 扫描已有课程，生成配置文件
/course scan <课程目录>         → 检查质量问题，输出报告
```

也可以在终端直接调用：

```bash
python3 scan.py init /path/to/course
python3 scan.py scan /path/to/course
```

## 五条检查规则

| 规则 | 检查内容 | 典型问题 |
|------|----------|----------|
| **新概念前置** | 首次引入概念时，深入内容前是否有铺垫 | 直接讲"闭包"没解释"函数作用域" |
| **代码可运行** | 代码示例复制粘贴后能否跑通 | 调用了未定义的函数、缺 import |
| **文档引用正确** | 图片路径、引用的文件名是否匹配实际文件 | 本地路径未替换为 CDN URL |
| **原理不跳过** | 关键结论是否不止给结果，还有解释 | 只贴了代码说"这样就对了"，没说为什么 |
| **概念衔接自然** | 章节之间有无过渡句 | 上一节讲 A，下一节直接讲 B |

## 八条生成规范

`design` 和 `lesson` 命令生成内容时自动遵循：

| # | 规范 | 说明 |
|---|------|------|
| 1 | 前置基础 | 每个新概念先铺垫再深入 |
| 2 | 代码可运行 | 所有示例完整可跑 |
| 3 | 图片 CDN | 本地图替换为 jsdelivr URL |
| 4 | 原理不跳过 | 关键结论配 2-3 句"为什么" |
| 5 | 章节过渡 | 节与节之间有一句过渡 |
| 6 | 上节课桥接 | 每课开头回链上节课内容 |
| 7 | 4 条总结 | 每课末尾 4 条编号要点 |
| 8 | 一个类比 | 每个抽象概念配一个白话类比 |

## 配置文件

```json
{
  "course_name": "课程名称",
  "concept_keywords": "核心概念关键词",
  "skip_patterns": ["阶段复习", "总结", "考核"],
  "doc_pattern": "课时*.md",
  "concept_map": { "闭包": "函数作用域" }
}
```

`init` 命令自动生成，可手动调整。


## 修补指南

| 问题 | 修复 |
|------|------|
| 前置基础缺失 | 在深入章节前插入 100-200 字概念铺垫，用比喻解释 |
| 代码不可运行 | 补全 import、把被调用函数定义移到调用之前 |
| 原理过简 | 代码块后补 2-3 句——"为什么这样做能解决问题" |
| 衔接断裂 | 两节间加过渡句——"A 有什么局限，所以需要 B" |
| 文档引用 | 替换本地路径为 jsdelivr CDN URL |
