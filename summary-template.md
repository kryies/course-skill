# 总结课结构模板

用于 Day/阶段总结课。和普通课时的区别：不做新概念铺垫，而是横向汇总。

## 结构顺序

```
## 一、上节课做了什么
    → 回链该阶段最后一课的具体内容

## 二、本阶段全景
    → 表格：每课标题 | 核心函数/概念 | 一句话结论

## 三、共同模式
    → 把这些课的共同规律提炼出来
    → "X 类问题都可以归结为 Y 种模式"

## 四、一个统一类比
    → 用一个生活类比串联本阶段的全部内容

## 五、从本阶段到下一阶段
    → 下一阶段学什么，为什么这个顺序

## 六、总结
    → 4 条编号要点
```

## 质量标准

- 每课至少有函数名回链（不是"第一课学了X"，是"`os.popen()` 把字符串交给 Shell"）
- 统一类比能覆盖全部课时
- 有向前看的过渡（"接下来为什么学这个"）
- 总结课不是目录——是把分散知识点串成一条线
