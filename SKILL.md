# Course Review — 课程设计与审查

设计新课程、生成课时文档、扫描已有课程的质量问题。不限于任何技术栈——Python、Java、数据科学、前端、安全均可。

## 命令

```
/course design <课程名>        → 交互式设计课程大纲 → 生成 课程设计.md + course-config.json
/course lesson <课程目录> <N>   → 生成/重写第 N 课（参考相邻课时风格和深度）
/course init <课程目录>         → 扫描已有课程，生成配置文件
/course scan <课程目录>         → 扫描质量问题，输出报告
```

## design —— 从零规划一门课

Claude 通过对话确认：课程主题、总课时、目标学生、前置要求、理论/实操比例。然后生成：

1. **课程设计.md** — 五天（或自定义周期）大纲，每天主题、每课标题、类型（理论/实操）、核心内容要点
2. **course-config.json** — 课程配置文件

design 阶段只规划到课时标题和内容要点，具体文档由 `lesson` 命令逐课生成。

## lesson —— 逐课生成文档

根据大纲生成/重写单课。生成时自动遵循八条生成规范，参考已有课时保持风格一致。

```
/course lesson <课程目录> 5     → 生成第 5 课
/course lesson <课程目录> 5-8   → 生成第 5-8 课
```

## init / scan —— 扫描已有课程

```
/course init <课程目录>    → 自动分析课时内容，生成 course-config.json
/course scan <课程目录>    → 按五条规则检查所有课时，输出问题报告
```

## 五条检查规则（scan 用 —— 通用版）

| 规则 | 检查内容 | 典型问题 |
|------|----------|----------|
| **新概念前置** | 首次引入概念时，深入内容前是否有铺垫 | 直接讲"闭包"没解释"函数作用域" |
| **代码可运行** | 代码示例复制粘贴后能否跑通 | 调用了未定义的函数、缺 import、依赖未声明 |
| **文档引用正确** | 图片路径、引用的文件名是否匹配实际文件 | 引用了本地路径未替换为 CDN URL |
| **原理不跳过** | 关键结论是否不止给结果，还有解释 | 只贴了代码说"这样就对了"，没说为什么 |
| **概念衔接自然** | 章节之间有无过渡句 | 上一节讲 A，下一节直接讲 B，无"为什么需要 B" |

## 八条生成规范（design / lesson 用）

生成课时文档时自动内置以下规范：

| # | 规范 | 说明 |
|---|------|------|
| 1 | **前置基础** | 每个新概念先铺垫再深入——用白话解释"是什么、解决什么问题" |
| 2 | **代码可运行** | 所有示例完整可跑——import 齐全、函数定义在调用之前、依赖明确 |
| 3 | **图片 CDN** | 本地图替换为 jsdelivr URL |
| 4 | **原理不跳过** | 关键结论配 2-3 句"为什么"——不只是"怎么做"，还有"为什么这样能解决" |
| 5 | **章节过渡** | 节与节之间有一句过渡——"A 有什么局限，所以需要 B" |
| 6 | **上节课桥接** | 每课开头一句话回链上节课——"上节课我们学了 X，这节课看 Y" |
| 7 | **4 条总结** | 每课末尾 4 条编号要点——覆盖本课最核心的结论 |
| 8 | **一个类比** | 每个抽象概念配一个白话类比——用日常生活经验解释技术概念 |

这些规范不涉及任何特定技术栈。教 Python 安全、Java 审计、数据科学、前端开发都能用。

## 黄金模板

生成时参考 skill 内置模板文件：

- `lesson-template.md` — 概念课/实操课标准结构（前置基础→正常用法→问题→根因→修复→总结）
- `summary-template.md` — 总结课结构（全景汇总→共同模式→统一类比→阶段过渡→总结）
- `config-template.json` — 配置文件格式参考

生成新课时自动匹配对应类型的结构和深度。

## 配置文件格式

```json
{
  "course_name": "课程名称",
  "concept_keywords": "核心概念关键词（用于判断新概念首次出现）",
  "skip_patterns": ["阶段复习", "总结", "考核", "CTF"],
  "doc_pattern": "课时*.md",
  "concept_map": {
    "闭包": "函数作用域",
    "原型链": "JavaScript继承"
  }
}
```

字段说明：
- `concept_keywords`：课程核心概念关键词，用 `|` 分隔。用于规则 1（新概念前置）判断——首次出现这些词时检查是否有铺垫段
- `skip_patterns`：文件名匹配这些模式的课时跳过检查（复习课、总结课等）
- `concept_map`：概念关键词 → 概念显示名。从课时内容自动提取，可手动调整

## 修补指南

| 问题类型 | 修复方法 |
|----------|----------|
| 前置基础缺失 | 在深入章节前插入 100-200 字概念铺垫，用比喻解释 |
| 代码不可运行 | 补全 import、把被调用函数定义移到调用之前、注明依赖 |
| 原理过简 | 代码块后补 2-3 句核心思路——"为什么这样做能解决问题" |
| 衔接断裂 | 两节间加过渡句——"A 有什么局限，所以需要 B" |
| 文档引用 | 替换本地路径为 jsdelivr CDN URL 或更正文件名 |
