# 课时结构模板

生成新课或审查已有课时时的参考标准。不涉及特定技术栈。

## 结构顺序

```
## 一、上节课做了什么
    → 一句话回链上节课的具体函数/概念

## 二、前置基础：X 是什么
    → 新概念先用白话解释"是什么、解决什么问题"
    → 配一个生活类比
    → 再给最小可运行代码示例

## 三、正常用法
    → 先展示正确的、安全的用法
    → 代码完整可跑（import 齐全、函数定义在调用之前）

## 四、问题/攻击/误区
    → 展示错误用法
    → 演示问题发生的过程
    → 代码可复现

## 五、为什么出问题——根因
    → 不是"这里写错了"，是"这里的设计假设被打破了"
    → 用一两句话说清因果链

## 六、正确做法
    → 修复代码
    → 2-3 句解释"为什么这样能解决问题"
    → 核心：不只是怎么做，更是为什么

## 七、总结
    → 4 条编号要点
    → 覆盖本课最核心的结论
```

## 质量标准

- 每段代码复制粘贴到干净环境能直接跑
- 抽象概念至少配一个白话类比
- 章节之间有过渡句（"A 有什么局限，所以需要 B"）
- 第一个代码示例应该展示正常用法，不是错误用法
- 防御/修复段落：代码 + 为什么能防住（缺一不可）
